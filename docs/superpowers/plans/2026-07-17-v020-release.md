### 后端

- `services/state_machine.rs::transition()`（L38）— 零调用方，所有真实调用走 `transition_with_dirs`
- `services/state_machine.rs::non_deleted_statuses()`（L131）— 零调用方

## 3. 代码解耦

### 抽公共 utility

- `formatSize(bytes)` 在 4 处实现相同 KB/MB 转换（`FileCard.vue:24` / `RecycleBinView.vue:67` / `SettingsView.vue:165` / `DirtyView.vue:10`）→ 抽 `src/lib/format.ts::formatBytes`
- `fmtMtime`（`SettingsView:173`）与 `toLocaleString`（`DirtyView:60`）— 实现不同但语义重叠；DirtyView 故意用本地化格式属于产品意图，不强求合并，**保留差异**

### 收口重复类型 / 标签

- `TagType` 类型在 `FileCard.vue:46` 与 `DetailView.vue:230` 各写一份 6 元素 union → 用 naive-ui 的 `TagProps['type']` 或抽 `src/lib/tag.ts`
- `statusLabel` / `fileStateLabel` 在 `FileCard.vue` 与 `DetailView.vue` 双份实现 → 抽 `src/lib/file-state.ts`（V4 业务状态的唯一中文权威）

### 收口后端双入口

- `dirty_scanner.rs:127-134` 内联 `match status → dir` → 替换为调用 `state_machine` 的 `expected_dir_for_status`（但该 helper 本身在删除清单——需要**决策**：是先统一再删 helper，还是直接保留 dirty_scanner 的内联 match、删 helper）→ **保留内联 match，删 helper**（dirty_scanner 的 match 已经稳定，少一个跨文件耦合更好）
