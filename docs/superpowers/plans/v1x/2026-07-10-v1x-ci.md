# V1.x Sub-Plan 8 — CI

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Implements umbrella candidate **#8**.

**Goal:** Add a GitHub Actions workflow that runs `cargo test` and `pnpm install && pnpm build` on every push/PR, so regressions in the new http integration tests from sub-plan 1 are caught before they ship.

**Architecture:** One workflow file at `.github/workflows/ci.yml`. Single job `ci` with two steps; matrix can be added later if Linux runners are needed.

**Tech Stack:** GitHub Actions `windows-latest`, the existing Rust + pnpm toolchain.

---

## Task 1: Workflow file

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Write the workflow**

```yaml
name: ci

on:
  push:
    branches: [master]
  pull_request:
    branches: [master]

jobs:
  rust:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          toolchain: stable
      - name: cache cargo
        uses: actions/cache@v4
        with:
          path: |
            ~/.cargo/registry
            ~/.cargo/git
            src-tauri/target
          key: cargo-${{ hashFiles('src-tauri/Cargo.lock', 'src-tauri/Cargo.toml') }}
      - run: cd src-tauri && cargo test --offline --no-run
        # `--no-run` for the first run so we don't fail when network is
        # blocked; subsequent runs can drop this flag.
      - run: cd src-tauri && cargo test --offline

  frontend:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
        with:
          version: 9
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm lint
      - run: pnpm build
```

- [ ] **Step 2: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: rust test + frontend lint/build on push"
```

## Task 2: Validate locally

- [ ] **Step 1: Reproduce the CI steps on the dev box**

Run (in order):
```bash
cd src-tauri && cargo test --offline
pnpm install --frozen-lockfile
pnpm lint
pnpm build
```
Expected: all four commands exit 0.

- [ ] **Step 2: Push and watch the run**

Run: `git push origin master`
Open the Actions tab on github.com and confirm both jobs pass.

- [ ] **Step 3: Fix any failure**

Common failures and fixes:
- **pnpm install cannot find lockfile** → ensure `pnpm-lock.yaml` is committed.
- **cargo build pulls deps** → ensure the cache key includes `Cargo.lock`.
- **frontend lint fails on existing warnings** → either fix the warning or add `pnpm lint --no-warn-ignored` to the workflow (only if the warning is pre-existing and benign).
